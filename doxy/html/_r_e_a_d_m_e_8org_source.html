1 #+TITLE: Wire Cell Toolkit
2 
3 This package provides a "build aggregation" package for various Wire Cell Toolkit (WCT) packages.
4 
5 The home page for Wire Cell is http://www.phy.bnl.gov/wire-cell/.
6 
7 * Installation
8 
9 Note: no particular environment variable settings are needed for
10 installation of the Wire Cell Toolkit and in fact are discouraged.
11 
12 ** Externals
13 
14 The core of WCT depends on a few packages. ROOT is needed for unit tests and some I/O related WCT packages.
15 
16 You may provide the necessary externals as you wish and various methods are outlined:
17 
18 - [[https://github.com/WireCell/wire-cell-spack][Spack-based install]] automatically builds all (non-OS) externals and WCT itself
19 - Some WCT releases are built at FNAL as a UPS product named =wirecell=.
20 
21 ** Developer Source
22 
23 Developers check out =master= branch via SSH.
24 
25 #+BEGIN_EXAMPLE
26  $ git clone --recursive git@github.com:WireCell/wire-cell-build.git wct
27 #+END_EXAMPLE
28 
29 ** User Source
30 
31 Users typically should build a release branch, either the tip or a
32 tagged release on that branch. Tagged releases are shown on the [[https://github.com/WireCell/wire-cell-build/releases][this
33 GitHub release page]]. To get the source, for example for release 0.5.0
34 on branch 0.5.x do:
35 
36 #+BEGIN_EXAMPLE
37  $ git clone --recursive --branch 0.5.x https://github.com/WireCell/wire-cell-build.git
38  $ git checkout -b 0.5.0 0.5.0
39  $ git submodule init
40  $ git submodule update
41  $ git submodule foreach git checkout -b 0.5.0 0.5.0
42 #+END_EXAMPLE
43 
44 Anonymous users can clone the =master= branch but will need to swith the submodule urls to use HTTPS. A script is provided to make that easier:
45 
46 #+BEGIN_EXAMPLE
47  $ git clone https://github.com/WireCell/wire-cell-build.git
48  $ ./swith-git-urls
49  $ git submodule init
50  $ git submodule update
51 #+END_EXAMPLE

53 ** Configuring the source
54 
55 On well-behaved systems you can do:
56 
57 #+BEGIN_EXAMPLE
58  $ ./wcb configure --prefix=/path/to/install
59 #+END_EXAMPLE
60 
61 If your system is not well-behaved you can exhaustively specify where externals are found. In this contrived example, they are installed all into one directory named by the =WCT_EXTERNALS= variable (such as when making a Spack "view")
62 
63 #+BEGIN_EXAMPLE
64  $ ./wcb --help
65  $ ./wcb configure --prefix=/path/to/install \
66  --boost-includes=$WCT_EXTERNALS/include --boost-libs=$WCT_EXTERNALS/lib --boost-mt \
67  --with-eigen=$WCT_EXTERNALS --with-jsoncpp=$WCT_EXTERNALS --with-tbb=$WCT_EXTERNALS \
68  --with-root=$WCT_EXTERNALS --with-fftw=$WCT_EXTERNALS --with-root=$WCT_EXTERNALS
69 #+END_EXAMPLE
70 
71 ** Building
72 
73 To build the code but not run the tests do
74 
75 #+BEGIN_EXAMPLE
76  $ ./wcb --notests
77 #+END_EXAMPLE
78 
79 ** Installing
80 
81 To install the code but not run the tests do:
82 
83 #+BEGIN_EXAMPLE
84  $ ./wcb --notests install
85 #+END_EXAMPLE
86 
87 ** Testing
88 
89 To run the tests:
90 
91 #+BEGIN_EXAMPLE
92  $ ./wcb --alltests --testcmd="env LD_LIBRARY_PATH=$WCT_EXTERNALS/lib:`pwd`/install/lib %s"
93 #+END_EXAMPLE
94 
95 In principle, no tests should fail but lazy developers being as they are, some may. Typically more than 10% indicate something more wrong than just laziness. Feel free to gripe at the developers.

97 * Release management
98 
99 To make releases, the above details are baked into two test scripts [[https://github.com/WireCell/waf-tools/blob/master/make-release.sh][make-release.sh]] and [[https://github.com/WireCell/waf-tools/blob/master/test-release.sh][test-release.sh]]. See comments at the top of each for how to run them. These scripts can be used by others but are meant for developers to make official releases.
