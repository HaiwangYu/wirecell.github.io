21 ** External software dependencies
22 
23 The core of WCT depends on:
24 
25 - Boost
26 - JsonCPP
27 - Jsonnet
28 - Eigen
29 - FFTW
30 
31 Optional dependencies include
32 
33 - ROOT
34 - TBB
35 
36 You may provide the necessary external software dependencies as you
37 wish. Some options include:
38 
39 - [[https://github.com/WireCell/wire-cell-spack][Spack-based install]] automatically builds all (non-OS) externals and WCT itself
40 - Some WCT releases are built at FNAL as a UPS product named =wirecell=.
41 - Exploit the above with a [[https://github.com/WireCell/wire-cell-singularity][Singularity container and CVMFS]] (currently recommended)

51 ** User Source
52 
53 Users typically should build a release branch, either the tip or a
54 tagged release on that branch. Tagged releases are shown on the [[https://github.com/WireCell/wire-cell-build/releases][this
55 GitHub release page]]. To get the source, for example for release 0.5.0
56 on branch 0.5.x do:
57 
58 #+BEGIN_EXAMPLE
59  $ git clone --recursive --branch 0.5.x https://github.com/WireCell/wire-cell-build.git
60  $ git checkout -b 0.5.0 0.5.0
61  $ git submodule init
62  $ git submodule update
63  $ git submodule foreach git checkout -b 0.5.0 0.5.0
64 #+END_EXAMPLE
65 
66 Anonymous users can clone the =master= branch but will need to switch
67 the submodule urls to use HTTPS. A script is provided to make that
68 easier:
69 
70 #+BEGIN_EXAMPLE
71  $ git clone https://github.com/WireCell/wire-cell-build.git
72  $ ./swith-git-urls
73  $ git submodule init
74  $ git submodule update
75 #+END_EXAMPLE
76 
77 Except when mistakes are made, the release branches default to
78 anonymous HTTPS URLs.

80 ** Configuring the source
81 
82 On well-behaved systems you can do:
83 
84 #+BEGIN_EXAMPLE
85  $ ./wcb configure --prefix=/path/to/install
86 #+END_EXAMPLE
87 
88 If your system is not well-behaved you can exhaustively specify where externals are found. In this contrived example, they are installed all into one directory named by the =WCT_EXTERNALS= variable (such as when making a Spack "view")
89 
90 #+BEGIN_EXAMPLE
91  $ ./wcb --help
92  $ ./wcb configure --prefix=/path/to/install \
93  --boost-includes=$WCT_EXTERNALS/include --boost-libs=$WCT_EXTERNALS/lib --boost-mt \
94  --with-eigen=$WCT_EXTERNALS --with-jsoncpp=$WCT_EXTERNALS --with-tbb=$WCT_EXTERNALS \
95  --with-root=$WCT_EXTERNALS --with-fftw=$WCT_EXTERNALS --with-root=$WCT_EXTERNALS
96 #+END_EXAMPLE
97 
98 To force no use of ROOT, even if it is found, use ~--with-root=false~.
99

101

103 
104 #+BEGIN_EXAMPLE
105  $ ./wcb -p --notests
106 #+END_EXAMPLE

109

111 
112 #+BEGIN_EXAMPLE
113  $ ./wcb -p --notests install
114 #+END_EXAMPLE
115 
116 ** Testing
117 
118 To run the tests, depending on your installation environment you may
119 need to set ~LD_LIBRARY_PATH~ and will need to set ~WIRECELL_PATH~ to find
120 both ~data~ and ~cfg~ directories.
121 
122 #+BEGIN_EXAMPLE
123  $ export LD_LIBRARY_PATH=/path/to/install/lib:$LD_LIBRARY_PATH
124  $ export WIRECELL_PATH=/path/to/data:/path/to/cfg
125 #+END_EXAMPLE
126 
127 Then,
128 
129 #+BEGIN_EXAMPLE
130  $ ./wcb -p --alltests
131 #+END_EXAMPLE
132 
133 In principle, no tests should fail.
